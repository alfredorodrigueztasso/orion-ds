# Getting Started

Welcome to Orion Design System ‚Äî an **AI-first design system** built on the **Chain of Truth** principle. This guide will get you up and running in minutes.

## What is Orion?

Orion is a comprehensive, production-ready component library for building web applications with:

- **69+ components** ‚Äî Buttons, cards, modals, forms, tables, charts, and more
- **44 pre-built sections** ‚Äî Hero, features, pricing, team, testimonials, and other full-page blocks
- **12 complete templates** ‚Äî Landing pages, dashboards, dashboards, login pages, and more
- **Multi-brand support** ‚Äî Switch between orion, red, deepblue, and orange brands with zero changes to component code
- **Three visual modes** ‚Äî Display (marketing), Product (dashboards), and App (mobile-first) for different contexts
- **Tree-shakeable** ‚Äî Import only what you use, keep bundles small
- **Type-safe** ‚Äî Full TypeScript support with auto-generated types

## The Chain of Truth

Orion eliminates UI hallucination by enforcing strict separation between **primitives**, **semantics**, and **components**:

```
Layer 1: Primitives (source truth)
‚îú‚îÄ Colors: #1B5BFF, #FFFFFF, #000000, etc.
‚îú‚îÄ Spacing: 4px, 8px, 12px, 16px, etc.
‚îú‚îÄ Typography: Libre Baskerville, DM Sans, JetBrains Mono
‚îî‚îÄ Radius: 6px, 12px, 16px, 9999px

      ‚Üì (aliased as)

Layer 2: Semantics (intent-based tokens)
‚îú‚îÄ --interactive-primary ‚Üí uses brand color
‚îú‚îÄ --surface-base ‚Üí uses light/dark background
‚îú‚îÄ --text-secondary ‚Üí uses contrast-adjusted text
‚îî‚îÄ --radius-control ‚Üí uses radius scale

      ‚Üì (consumed by)

Layer 3: Components (blind consumers)
‚îú‚îÄ Button: uses --interactive-primary, --radius-control, --font-secondary
‚îú‚îÄ Card: uses --surface-base, --border-subtle
‚îî‚îÄ All components automatically adapt to brand/theme/mode changes
```

## Setup: ThemeProvider

Wrap your app with the `ThemeProvider` at the root level. This enables:
- Brand and theme switching
- Dark/light mode persistence
- Global style management

**React:**
```tsx
import { ThemeProvider } from '@orion-ds/react/client';
import '@orion-ds/react/styles.css';

export default function App() {
  return (
    <ThemeProvider>
      <YourContent />
    </ThemeProvider>
  );
}
```

**Next.js (App Router):**
```tsx
import { ThemeProvider } from '@orion-ds/react/client';
import '@orion-ds/react/styles.css';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ThemeProvider>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}
```

Fonts load **automatically** ‚Äî no manual Google Fonts setup needed.

All components automatically inherit:

**No brand prop needed** ‚Äî brand is global.

## Multi-Brand Support

Orion ships with 4 built-in brands:

| Brand | Accent Color | Radius | Use Case |
|-------|-------------|--------|----------|
| **orion** | Blue (#1B5BFF) | 12px | Default, professional |
| **red** | Red (#D7282F) | 9999px (pill) | Bold, energetic |
| **deepblue** | Deep Blue (#006FBA) | 12px | Conservative, corporate |
| **orange** | Red-Orange (#F15D22) | 9999px (pill) | Warm, modern |

Switch brands at runtime using the `useThemeContext()` hook or manually setting `data-brand`:

```tsx
import { useThemeContext } from '@orion-ds/react';

export function BrandSwitcher() {
  const { brand, setBrand } = useThemeContext();

  return (
    <div>
      <p>Current brand: {brand}</p>
      <button onClick={() => setBrand('red')}>Switch to Red</button>
      <button onClick={() => setBrand('orion')}>Back to Orion</button>
    </div>
  );
}
```

## Dark Mode

Orion supports light and dark themes out of the box. Switch themes using `useThemeContext()`:

```tsx
import { useThemeContext } from '@orion-ds/react';

export function ThemeToggle() {
  const { theme, setTheme } = useThemeContext();

  return (
    <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
      {theme === 'light' ? 'üåô Dark Mode' : '‚òÄÔ∏è Light Mode'}
    </button>
  );
}
```

The theme preference is **automatically persisted** to localStorage.
