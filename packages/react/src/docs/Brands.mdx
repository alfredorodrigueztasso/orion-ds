## Setting Up ThemeProvider

Wrap your app with `ThemeProvider` and optionally set a default brand:

```tsx
import '@orion-ds/react/styles.css';
import { ThemeProvider } from '@orion-ds/react';

function App() {
  return (
    <ThemeProvider defaultBrand="deepblue" defaultTheme="light">
      <YourApp />
    </ThemeProvider>
  );
}
```

<Callout variant="tip" title="Fonts load automatically">
  <code>ThemeProvider</code> detects the active brand and loads the correct Google Fonts. No manual <code>&lt;link&gt;</code> tags required.
</Callout>

## Switching Brand and Theme

Use the `useTheme` hook to control brand and theme programmatically:

```tsx
import { useTheme } from '@orion-ds/react';

function BrandSwitcher() {
  const { theme, brand, setTheme, setBrand, toggleTheme } = useTheme();

  return (
    <div>
      <p>Current: {brand} / {theme}</p>

      {/* Toggle light/dark */}
      <button onClick={toggleTheme}>
        Switch to {theme === 'light' ? 'dark' : 'light'}
      </button>

      {/* Switch brand */}
      <button onClick={() => setBrand('red')}>Red Brand</button>
      <button onClick={() => setBrand('deepblue')}>Deepblue Brand</button>
      <button onClick={() => setBrand('orange')}>Orange Brand</button>
      <button onClick={() => setBrand('lemon')}>Lemon Brand</button>
    </div>
  );
}
```

## How It Works

1. `ThemeProvider` sets `data-theme` and `data-brand` attributes on `<html>`
2. CSS uses selectors like `[data-brand="deepblue"]` to override token values
3. Components use semantic tokens (`var(--interactive-primary)`) that resolve per-brand
4. Changing the brand attribute instantly updates every component

```html
<!-- Light theme, Orion brand (default) -->
<html data-theme="light">

<!-- Dark theme, Deepblue brand -->
<html data-theme="dark" data-brand="deepblue">
```

In addition to brands and themes, Orion has a **mode** system that controls visual behavior:

  </FeatureCard>

Set the mode via data attribute or ThemeProvider:

```html
<html data-theme="light" data-brand="orion" data-mode="product">
```

Components use mode-aware tokens automatically:

```css
/* These tokens change per mode */
var(--mode-shadow-hover)     /* Shadow on hover */
var(--mode-hover-lift)       /* Transform on hover */
var(--mode-hover-duration)   /* Animation speed */
```
